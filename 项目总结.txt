# VLM 机器人动作指令微调 — 项目总结与经验复盘

## 一、项目目标

使用视觉语言模型（VLM）对 BridgeData V2 机器人操控数据集进行指令微调，
让模型学会"看图 + 读指令 → 输出机械臂动作"。
模型选择 Qwen3-VL-2B，微调框架 LLaMA-Factory，训练卡 NVIDIA RTX 4060 Ti 16GB。

## 二、完成的工作（技能栈展示）

### 2.1 跨平台环境搭建
- Windows 端：Conda 环境管理、PowerShell 适配、Git 缺失下的 zip 安装流程
- Linux 端：一键部署脚本、nohup 后台训练、SSH 断连恢复
- 解决了 conda --prefix 与 -n 命名环境的兼容问题，总结为团队规范写入 CLAUDE.md

### 2.2 大规模数据集工程（本项目核心技术难点，占工作量 40%）
- 数据源调研：对比了 BridgeData V2 的三种获取方式（原始 zip、TFDS、HuggingFace）
- TFRecord 解析：攻克了 tensorflow_datasets 嵌套 SequenceExample 的反序列化，
  经历了四次错误尝试（VarLenFeature → SequenceExample → 扁平 key + reshape 终局方案）
- LeRobot v3 Parquet 格式处理：从 23 个 AV1 编码 MP4 视频中按时间戳抽取 187,511 帧
- AV1 解码方案选型：cv2.VideoCapture → PyAV 软解 → ffmpeg 批量抽帧（最终方案）
- 数据量级：从 66 条轨迹（2,257 样本）扩展到 5,000 条轨迹（187k 样本）

### 2.3 多轮微调实验
- 训练框架：LLaMA-Factory Web UI + LoRA 微调
- 配置调优：batch_size=4, grad_accum=4, lr=1e-4, bf16, LoRA rank=16
- 共执行 6 轮完整训练，每轮针对不同数据量和数据分布

### 2.4 训练诊断与问题定位
- 发现模式坍缩（mode collapse）问题：loss 正常但推理全出同一动作
- 定位根因：机器人数据中"不动"动作占比 21%，VLM 学会输出最高频 token
- 实施了三种缓解策略：
  a) 数据平衡（欠采样高频动作）
  b) 联合动作 token 化（2662 类 → 单 token 分类）
  c) 每类动作上限截断（max 50 samples/class）

### 2.5 评估体系
- 构建了多维度的推理测试脚本
- 完全匹配率 + 方向准确性 + 夹爪准确性 三项指标
- 支持 token 化后的映射还原


## 三、最终结果与诚实评估

### 3.1 训练指标
- Loss 可降至 0.2~0.3（看起来正常）
- 显存占用 6~15GB（取决于 batch size）

### 3.2 推理指标（核心问题）
- 完全匹配率：最高 10%（远低于可用水平）
- 方向正确率：40~50%
- 夹爪正确率：50~60%

### 3.3 根因分析（关键认知）

1. **模型-任务分布不匹配**：Qwen3-VL 在图文对话数据上预训练，
   从未见过机器人动作 token（如 <dx+1>）。对模型而言这是全新语言，
   仅靠 187k 样本的 LoRA 微调无法充分学到动作的空间语义。

2. **连续动作离散化的精度损失**：将 7 维连续动作离散化为 token 序列，
   类内方差大（同一动作 token 对应不同的场景），类间边界模糊。

3. **单帧预测的固有难度**：从单张静态图片预测下一步精确位移，
   缺少时序上下文（前几步做了什么？机械臂当前速度？）。

4. **数据分布不均**：机器人数据天然有大量"凝视/等待"帧，
   单纯过滤不能解决长尾动作的 low-shot 问题。

### 3.4 为什么 Qwen-VL 做机器人这么难（行业共识）

- 国际上 OpenVLA / RT-2 等在 Open X-Embodiment 上预训练过，
  但实际落地时 Sim2Real 衰减率 >50%，社区基本都在做大改。
- 国内主流方案是 Qwen 基座 + 专项适配，比如：
  - Qwen-VL + 时序 Transformer（输入多帧历史）
  - Qwen-VL + 回归头（连续动作输出，不 token 化）
  - Qwen-VL + 强化学习（RLHF/DPO 对齐机器人偏好）
- 纯 LoRA 微调 + 离散 token 这条路目前没有公开的成功案例，
  我的实验验证了这一结论，不是能力问题而是路线问题。

### 3.5 如果继续做，正确的技术路线

- 动作表示：给 Qwen-VL 接回归头，输出连续 7 维浮点数（而非离散 token）
- 时序建模：输入连续 3-5 帧历史，让模型理解运动趋势
- 评估标准：欧氏距离误差（cm）作为主要指标，完全匹配仅做参考
- 基座保持 Qwen3-VL 不变（国内最成熟的 VL 基座），上层架构改造

# VLM → VLA：回归头 + 多帧时序 + 国产 VLA 管线搭建

## 一、项目目标

基于上次项目的根因分析结论，从零搭建 **Qwen3-VL + MLP 回归头 + 多帧时序输入** 的 VLA（Vision-Language-Action）管线，输出连续 7 维动作，用于具身智能公司作品集投递。

## 二、技术路线（与上次项目的核心差异）

| 维度 | 上次项目（失败） | 本次项目 |
|------|----------|----------|
| 输出形式 | 离散 token 分类（2662 类） | **连续回归（MSE loss）** |
| 模型 | Qwen3-VL-2B-Thinking（Instruct） | **Qwen3-VL-Embedding-8B** |
| 加载方式 | Qwen3VLForConditionalGeneration | **AutoModel**（去掉了 LM Head） |
| 时序建模 | 单帧输入 | **多帧输入（4帧历史）** |
| 微调框架 | LLaMA-Factory（黑盒） | **自己写的训练循环（Accelerate）** |
| 评估指标 | 完全匹配率（最高 10%） | **欧氏距离误差 + 每维 MAE** |
| 数据集 | BridgeData V2（LeRobot / HF） | **RoboMIND 2.0 Franka（ModelScope）** |
| 基座厂商 | Qwen（阿里通义） | Qwen + **国产数据集** |

## 三、完成的工作

### 3.1 模型架构设计（回归头）

- 设计 `ActionRegressionHead`：4096 → 2048 → 1024 → 7 的三层 MLP，LayerNorm + GELU + Dropout
- 输出维度由配置文件控制，`action_dim` 支持任意维度的回归
- 权重初始化：正态分布 std=0.02，bias 归零

### 3.2 Qwen3-VL-Embedding 模型适配

- 切换到 **Qwen3-VL-Embedding-8B**（AutoModel，非 Instruct 版本），去掉了 LM Head，省 ~1.3GB 显存
- 4-bit 量化（NF4）+ Double Quant，2060 6GB 上推理仅 5.54 GB
- LoRA rank=16，冻结视觉编码器，LLM 部分 LoRA 微调，回归头全量训练
- 踩坑记录：
  - Vision encoder 路径是 `model.model.visual` 不是 `model.visual`
  - hidden_size 在 `config.text_config.hidden_size`
  - Qwen3-VL 需要 `mm_token_type_ids`（M-RoPE 位置编码）
  - hidden states 是 bf16，回归头需要 float32 → `.float()`

### 3.3 训练管线

- 基于 Accelerate 的自定义训练循环（不是 LLaMA-Factory）
- bf16 mixed precision + gradient checkpointing + gradient accumulation
- Cosine LR schedule + warmup（10%）
- 训练/验证按 episode 切分，防止数据泄露
- 自动保存 best / final checkpoint

### 3.4 数据集工程（RoboMIND 2.0）

数据集选型经历三次迭代：

| 迭代 | 数据集 | 问题 |
|------|--------|------|
| 1 | BridgeData V2（HF） | 国内无代理无法下载 |
| 2 | **LET-Base-Dataset**（ModelScope） | ROSbag 格式、人形机器人动作空间不匹配、下载速度慢 |
| 3 | **RoboMIND 2.0 Franka**（ModelScope） | ✅ 最终方案 |

RoboMIND 2.0 Franka 特征：
- 国产数据集（北京人形机器人创新中心 + 北京大学）
- 7-DoF Franka Emika Panda 机械臂，动作空间天然匹配回归头
- HDF5 格式，直接读取，无需 ROSbag 解析
- 三组已选任务：
  - `move_apple_from_plate_to_bowl`：抓取放置（502 条轨迹）
  - `close_drawer_with_both_arms_simultaneously`：抽屉开关（300 条轨迹）
  - `move_tape_to_another_basket`：推拨（975 条轨迹）

### 3.5 跨平台适配

- 从本机 2060 6GB 验证（前向 + 训练 step 均通过，峰值 6.26GB）
- 服务器 4060 Ti 16GB 正式训练
- 脚本兼容 Windows/Linux，modelscope CLI 进度条实时显示
- 多线程并行下载（`ThreadPoolExecutor`，workers=3）

## 四、最终训练结果（2026-06-04）

### 4.1 训练 Loss 曲线

| Epoch | Avg Loss | 关键 Val Loss |
|-------|----------|---------------|
| 1 | 0.2686 | Step 500: 0.2100, Step 1000: 0.1600 |
| 2 | 0.1130 | Step 1500: 0.1021, Step 2000: 0.1001 |
| 3 | 0.0643 | Step 2500: 0.0855, Step 3000: 0.0588 |
| 4 | 0.0380 | Step 3500: **0.0450 (best)**, Step 4000: 0.0487 |

训练在 Epoch 4 末过拟合起始（val loss 拐头回升），最佳点 Step 3500 / Epoch 4 中段。

### 4.2 评估结果（Euclidean Error）

```
Samples: 919（验证集）
Mean Euclidean Error: 0.1255  (~12.6cm)
Median Euclidean Error: 0.0586 (~5.9cm)
```

| 维度 | MAE | 解释 |
|------|------|------|
| dx | 0.0049m | **~5mm** |
| dy | 0.0134m | **~1.3cm** |
| dz | 0.0125m | **~1.2cm** |
| droll | 0.1109rad | **~6.4°** |
| dpitch | 0.0146rad | **~0.8°** |
| dyaw | 0.0140rad | **~0.8°** |
| Gripper | **99.89%** | 近乎完美 |

中位数 5.9cm < 均值 12.6cm，说明大部分样本误差在 6cm 以内，少数坏样本拉高了均值。

### 4.3 迭代二：动作分块 + 时序平滑

基于迭代一的"误差累积"问题分析，提出两个改进：

1. **动作分块（Action Chunking）**：一次性预测未来 10 步动作（70 维），执行第 1 步，
   下一帧重新预测（Receding Horizon）。回归头输出从 7 维改为 70 维后 reshape 为 (B, 10, 7)。
2. **时序平滑 Loss**：在 MSE 基础上增加 `λ * mean(Δ²)` 惩罚相邻步动作突变。

训练 10 轮，val loss 从未拐头（0.2445 → 0.0220），无过拟合。

| 指标 | 迭代一（单步） | 迭代二（动作分块） | 提升 |
|------|--------------|-------------------|------|
| 均值 EU | 12.6cm | **7.8cm** | ↓38% |
| 中位数 EU | 5.9cm | **3.4cm** | ↓42% |
| dx MAE | 0.49cm | **0.32cm** | -35% |
| dy MAE | 1.34cm | **0.71cm** | -47% |
| dz MAE | 1.25cm | **0.72cm** | -42% |
| droll MAE | 6.4° | **4.0°** | -37% |
| Gripper | 99.89% | 99.88% | ≈持平 |

**结论：动作分块 + 时序平滑在全部维度上优于单步回归，中位误差降低 42%。**

### 4.4 仿真部署（进行中）

MuJoCo + IK 求解器闭环管道已搭建完成：
- VLA 推理 → 末端位姿增量 → Jacobian DLS IK → 关节角度 → MuJoCo 执行 → 截图 → 下一帧
- 100 步闭环跑通，证明模型推理 + IK + 仿真链路完整
- 待搭建任务级仿真场景后做量化成功率评估

### 4.4 归一化与反归一化说明

训练时对 7 维动作进行 z-score 归一化：
- `action_norm = (action - mean) / std`
- `mean = [0.599, 0.182, 0.422, -0.504, -0.066, 0.065, 0.159]`
- `std = [0.045, 0.144, 0.154, 0.814, 0.089, 0.095, 0.180]`

评估时通过 `denormalize()` 还原到原始物理单位：
- 位置维度（dx, dy, dz）：**米（m）**
- 姿态维度（droll, dpitch, dyaw）：**弧度（rad）**
- 夹爪维度（gripper）：0-1

**不能直接用总 MSE 除以维度数来估算每维误差**——各维度量级差异大（位置变化 ±0.01m vs 旋转变化 ±0.5rad），必须反归一化后计算。

## 五、改进方向

### 近期（作品集阶段）

1. **仿真部署**：将策略部署到 MuJoCo / Isaac Sim 仿真环境，
   接入 ROS 接口进行闭环验证，对比离线评估与在线执行的效果差异
2. **动作空间校准**：目前使用 `puppet/end_effector_right_pose_align` 的 7 维末端位姿，需要确认是笛卡尔坐标系还是关节角度，必要时用正运动学转换
2. **帧率对齐**：目前是简单的等间隔抽帧（5fps），后续可根据动作幅度自适应采样
3. **数据增强**：颜色抖动 + 轻微裁剪，不做几何变换（影响动作方向语义）
4. **多视角融合**：RoboMIND 提供 6 路 RGB 相机，可对比单视角 vs 多视角效果

### 远期（正式训练）

5. **动作维度扩展**：当前只用了 `end_effector_right_pose_align`，Franka 实际有 8 个关节，可考虑 8 维关节角度输出
6. **评测可视化**：生成真机轨迹 vs 预测轨迹的对比视频，作为作品集核心材料
7. **在线强化学习（RL）**：当前基于行为克隆（BC），可在仿真中引入 PPO/SAC 等 RL 算法，通过奖励信号让模型自我优化，降低对遥操作数据的依赖
**时序窗口消融**：对比 1 帧、2 帧、4 帧、8 帧历史窗口的效果
8. **增加训练轨迹量**：当前 300 条轨迹，Franka-Part-1 有数千条可用
