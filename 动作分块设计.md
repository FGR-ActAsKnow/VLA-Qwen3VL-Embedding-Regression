# 动作分块（Action Chunking）设计方案

## 一、问题

当前单步回归：每帧预测 1 个动作（7 维）。仿真时有误差累积——
上一步偏差 → 下一步在偏差上预测 → 轨迹发散。

## 二、方案

一次性预测未来 N 步动作序列，执行第 1 步，下一帧用新观测重新预测。

```
旧：每帧输出 7 维 [dx, dy, dz, dr, dp, dy, grip]
新：每帧输出 70 维 [dx₁..grip₁, dx₂..grip₂, ..., dx₁₀..grip₁₀]
                   ←-- 第1步 --→  ←-- 第2步 --→     ←-- 第10步 --→
```

推理时只取第 1 步执行，下一帧重新预测（receding horizon）。

## 三、改动的文件

### 3.1 configs/config.yaml

```
data:
  action_dim: 7           → 70
  action_horizon: 1        → 10
training:
  temporal_smooth_weight: 0.01    # 新增：时序平滑系数 λ
```

### 3.2 src/model/regression_head.py

```python
# 输出层 7 → 70
nn.Linear(1024, action_dim * action_horizon)  # action_dim=7, action_horizon=10
# forward 后 reshape: (batch, 70) → (batch, 10, 7)
```

其他代码不需要改——回归头输出维度由配置文件自动适配。

### 3.3 src/model/qwen3vl_robot.py

forward 中 loss 改成：

```python
# 原始 MSE
mse_loss = F.mse_loss(action_pred, action_labels)

# 时序平滑：相邻步动作的差值平方和
# action_pred shape: (batch, horizon, dim)
diffs = action_pred[:, 1:, :] - action_pred[:, :-1, :]
smooth_loss = diffs.pow(2).mean()

# 总损失
loss = mse_loss + self.temporal_smooth_weight * smooth_loss
```

### 3.4 src/data/dataset.py

`__getitem__` 中，从当前帧开始取未来 10 帧的动作拼接为 70 维标签：

```python
# 当前样本是 window[-1]，取 window + 未来 9 帧的动作
actions = []
for i in range(action_horizon):  # 10
    if idx + i < len(self.windows):
        a = self.windows[idx + i][-1]["action"]
    else:
        a = [0.0] * 7   # 超出轨迹填充 0
    actions.extend(normalizer.normalize(a))
# actions: 70 维
```

### 3.5 src/train/trainer.py

初始化时从 config 读取 `temporal_smooth_weight`，传给 model。

## 五、仿真设计

训练完成后，搭建 MuJoCo 闭环：

```
仿真初始化 Franka 环境
加载 VLA 模型
while True:
    render 相机图像
    堆叠 4 帧历史
    模型推理 → 70 维动作块
    取前 7 维执行
    等待下一帧
```
